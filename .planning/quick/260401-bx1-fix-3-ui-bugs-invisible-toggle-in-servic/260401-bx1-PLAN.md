  <action>
In `components/dashboard-sidebar.tsx`, the nav items array (around line 87) has:
```ts
{ href: '/dashboard/settings', section: 'dashboard', tKey: 'settings', icon: Settings, exact: false },
{ href: '/dashboard/settings/billing', section: 'dashboard', tKey: 'billing', icon: CreditCard, exact: false },
```

The active check (line ~134) is:
```ts
const isActive = exact ? pathname === href : pathname.startsWith(href)
```

Because `exact: false` on `/dashboard/settings`, visiting `/dashboard/settings/billing` causes `pathname.startsWith('/dashboard/settings')` to return true for BOTH items.

Fix: Change the settings nav item to use `exact: true`:
```ts
{ href: '/dashboard/settings', section: 'dashboard', tKey: 'settings', icon: Settings, exact: true },
```

This makes the settings link active only when `pathname === '/dashboard/settings'` exactly, while the billing link remains `exact: false` so it matches `/dashboard/settings/billing` and any future sub-routes.
