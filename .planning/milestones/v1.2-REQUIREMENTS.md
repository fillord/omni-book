# Requirements: omni-book v1.2

**Defined:** 2026-03-19
**Core Value:** A reliable, correctly-rendered booking experience for tenants and customers — accurate data display, accessible UI across all screen sizes and both themes.

## v1.2 Requirements

### Duration Input

- [x] **DUR-01**: The `durationMin` Select dropdown in `service-form.tsx` is replaced with a number input — tenants can enter any integer value
- [x] **DUR-02**: Custom +/− stepper buttons flanking the input increment/decrement duration by 1 minute
- [x] **DUR-03**: A "min" suffix is visible inside the duration input widget
- [x] **DUR-04**: Three quick-select preset buttons (15, 30, 60) appear below the input and set the value directly
- [x] **DUR-05**: The input accepts any value from 1 to 1440 minutes (1 min → 24 hours)
- [x] **DUR-06**: Zod schema enforces min(1)/max(1440) bounds for `durationMin`; the `DURATION_OPTIONS` constant is removed

### Resource Types

- [x] **RES-01**: Medicine niche gains 6 new resource types: doctor, nurse, operating_room, laboratory, office, treatment_room (replaces 3 opaque opt_xxx types)
- [x] **RES-02**: Horeca (restaurant) niche gains 5 new resource types: table, vip_lounge, terrace, bar_counter, karaoke_room
- [x] **RES-03**: Sports niche gains 5 new resource types: court, field, pool, sauna, equipment
- [x] **RES-04**: Beauty niche gains 3 physical workspace types: manicure_table, sink, couch

### Specialization

- [x] **SPEC-01**: Beauty `specialization` attribute is converted from a select (6 opt_xxx options) to free-text; horeca and sports staff gain a new `attr_specialization` free-text field for backward compatibility with existing staff records

### Internationalization

- [x] **I18N-01**: All 19 new resource type labels and `attr_specialization` have complete RU/EN/KZ translations (60 new translation entries) using `resource_type_<value>` and `attr_<field>` key patterns — no opt_xxx prefix

## Out of Scope

| Feature | Reason |
|---------|--------|
| Automated visual regression screenshots | Deferred to v2 |
| Third-party component internals (recharts fills) | Not accessible via class replacement; deferred to v2 |
| `hover-glow` CSS var refactor | Deferred to v2 |
