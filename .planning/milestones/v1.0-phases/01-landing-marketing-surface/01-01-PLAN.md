**Test structure — one describe block per requirement:**

**LAND-01 (backgrounds):** For each component file in `components/landing/` (HeroSection.tsx, PricingCards.tsx, FeaturesGrid.tsx, NicheCards.tsx, StatsCounter.tsx, Testimonials.tsx, DemoSection.tsx, FadeIn.tsx), assert that the file does NOT contain bare `bg-white`, `bg-zinc-50`, `bg-zinc-100`, `bg-zinc-200`, `bg-slate-50`, `bg-slate-100`, `bg-slate-200` classes. Use regex: `/(?<![-/])\bbg-(white|zinc-(50|100|200)|slate-(50|100|200))\b/`. **Exception:** `Footer.tsx` is exempt from LAND-01 — its `bg-zinc-900` is an intentional fixed dark footer. Include a separate test that Footer.tsx contains a comment with the word "intentional".

**LAND-02 (dark headings):** Assert no file contains bare `text-zinc-900`, `text-slate-900`, `text-gray-900`. Regex: `/(?<!dark:)text-(zinc|slate|gray)-900\b/`.

**LAND-03 (muted text):** Assert no file contains bare `text-zinc-400`, `text-zinc-500`, `text-slate-500`, `text-gray-500` WITHOUT a `dark:` prefix. Regex: `/(?<!dark:)text-(zinc-(400|500)|slate-500|gray-500)\b/`. **Important:** `dark:text-zinc-400` is acceptable (it is a dark-mode override on a brand-accent element) but bare `text-zinc-400` is not.

**LAND-04 (borders):** Assert no file contains bare `border-zinc-200`, `border-zinc-300`, `border-zinc-800`, `border-gray-200`, `border-gray-300`. Regex: `/(?<!dark:)border-(zinc|gray)-(200|300|800)\b/`. **Exception:** `border-zinc-600` with `dark:` prefix in HeroSection is acceptable (will be replaced, but if found with dark: prefix it is a transitional state).

**LAND-05 (hover states):** Assert no file contains `hover:bg-zinc-*`, `hover:bg-gray-*`, `hover:bg-slate-*`, or `hover:text-zinc-*` (with dark: prefix variant `dark:hover:` also caught). Regex: `/(dark:)?hover:(bg|text)-(zinc|gray|slate)-\d+/`.

**LAND-06 (PricingCards specific):** Read PricingCards.tsx. Extract only the non-highlighted card section (everything outside the `bg-indigo-600` highlighted Pro card block). Assert it contains no bare zinc/slate/gray classes. Implementation approach: assert the entire file has no bare zinc/slate/gray text/border/bg classes (the highlighted Pro card uses indigo/white classes, not zinc/slate/gray, so a whole-file check is safe).

**LAND-07 (gradients):** Read HeroSection.tsx. Assert the file contains `dark:from-` (adaptive gradient has dark variant). Read FeaturesGrid.tsx. Assert the icon container either contains `dark:bg-indigo-` OR contains a comment with "intentional" near the `bg-indigo-50` usage.

**Each test should use `it.each` or individual `it` blocks per component file** for clear failure reporting.
