**Step 1: Create `lib/validations/booking.ts`**

Create Zod schema for manual booking input:

```typescript
import { z } from 'zod'

export const manualBookingSchema = z.object({
  date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, 'Invalid date format'),
  resourceId: z.string().min(1, 'Resource required'),
  serviceId: z.string().min(1, 'Service required'),
  startsAt: z.string().min(1, 'Start time required'),
  endsAt: z.string().min(1, 'End time required'),
  clientName: z.string().min(1, 'Client name required').max(100),
  clientPhone: z.string().min(7, 'Phone required'),
})

export type ManualBookingInput = z.infer<typeof manualBookingSchema>
```

**Step 2: Create `lib/actions/bookings.ts`**

Create the Server Action file with `'use server'` directive at top.

Implement `createManualBooking(data: ManualBookingInput)`:
1. Call `requireAuth()` to get session
2. Call `requireRole(session, ['OWNER', 'STAFF', 'SUPERADMIN'])` to enforce admin access
3. Extract `tenantId` from `session.user.tenantId!`
4. Parse input with `manualBookingSchema.parse(data)`
5. Normalize phone with `normalizePhone(parsed.clientPhone)` from `@/lib/utils/phone`
6. Use `basePrisma.$transaction` with `{ isolationLevel: 'Serializable' }`:
   a. Lock resource row: `await tx.$queryRaw\`SELECT id FROM "Resource" WHERE id = ${parsed.resourceId} FOR UPDATE\``
   b. Check collision: `await tx.booking.findFirst({ where: { resourceId: parsed.resourceId, status: { not: 'CANCELLED' }, startsAt: { lt: new Date(parsed.endsAt) }, endsAt: { gt: new Date(parsed.startsAt) } } })`
   c. If collision found, throw an Error with message 'Time slot conflict'
   d. Create booking: `tx.booking.create({ data: { tenantId, resourceId: parsed.resourceId, serviceId: parsed.serviceId, guestName: parsed.clientName, guestPhone: normalizedPhone, startsAt: new Date(parsed.startsAt), endsAt: new Date(parsed.endsAt), status: 'CONFIRMED', manageToken: null } })`
7. After transaction, call `revalidatePath('/dashboard/bookings')`
8. Return `{ success: true, bookingId: result.id }`
9. Wrap in try/catch — on error return `{ success: false, error: err.message }`

CRITICAL per RESEARCH.md:
- `manageToken: null` — admin booking, no client self-manage link (per CRM-B07)
- Do NOT call `createBooking()` from engine — it enforces MAX_ACTIVE_BOOKINGS_PER_PHONE limit (Pitfall 3)
- Do NOT generate randomUUID for manageToken
- Use `basePrisma` (not tenant-scoped prisma) for the transaction since admin may need cross-checks
