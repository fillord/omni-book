Create `app/api/manage/[token]/reschedule/route.ts` with a single `POST` handler.

This route is **auth-free** — do NOT import `getServerSession`, `requireAuth`, or any auth guard.

```typescript
import { NextRequest, NextResponse } from "next/server"
import { z } from "zod"
import { basePrisma } from "@/lib/db"
import { sendTelegramMessage } from "@/lib/telegram"
import { format } from "date-fns"
import { ru } from "date-fns/locale"

const bodySchema = z.object({
  startsAt: z.string().datetime(),
  endsAt: z.string().datetime(),
})
```

POST handler implementation:

1. **Parse body**: Validate request body with `bodySchema.safeParse()`. Return 400 on parse failure.

2. **Token lookup**: Same pattern as cancel route — `basePrisma.booking.findUnique({ where: { manageToken: token } })`. Include tenant relation: `include: { tenant: { select: { name: true, phone: true, telegramChatId: true, timezone: true } }, service: { select: { name: true } } }`. Return 404 if not found.

3. **Status check**: Verify `booking.status` is `CONFIRMED` or `PENDING`. Return 422 with message `"Запись уже отменена или завершена"` otherwise.

4. **4-hour rule re-check server-side**: Same cutoff calculation as cancel route — compute cutoff from booking's CURRENT `startsAt`, return 422 if `now >= cutoff`.

5. **Collision check in a Serializable transaction**: Wrap the update in `basePrisma.$transaction(async (tx) => { ... }, { isolationLevel: "Serializable" })`:

   a. Lock the resource row: `await tx.$queryRaw\`SELECT id FROM "Resource" WHERE id = ${booking.resourceId} FOR UPDATE\``

   b. Check for overlapping bookings on the same resource (exclude the current booking by id):
   ```typescript
   const collision = await tx.booking.findFirst({
     where: {
       tenantId: booking.tenantId,
       resourceId: booking.resourceId,
       id: { not: booking.id },
       status: { in: ["CONFIRMED", "PENDING"] },
       AND: [
         { startsAt: { lt: new Date(endsAt) } },
         { endsAt: { gt: new Date(startsAt) } },
       ],
     },
   })
   ```
   If collision found, return from the transaction with an error signal (throw or return null).

   c. Update booking in place:
   ```typescript
   return tx.booking.update({
     where: { id: booking.id },
     data: {
       startsAt: new Date(startsAt),
       endsAt: new Date(endsAt),
     },
   })
   ```

6. If collision was detected, return 409 with message `"Выбранное время уже занято"`.

7. **Telegram notification to tenant owner** (fire-and-forget):
   ```typescript
   const tenantChatId = (booking.tenant as { telegramChatId?: string | null })?.telegramChatId
   if (tenantChatId) {
     const fmt = (d: Date) => format(d, "d MMMM yyyy, HH:mm", { locale: ru })
     const msg = [
       "🔄 <b>Перенос записи!</b>",
       `👤 Клиент: ${booking.guestName} (${booking.guestPhone})`,
       `🛠 Услуга: ${booking.service?.name ?? ""}`,
       `📅 Было: ${fmt(booking.startsAt)}`,
       `📅 Стало: ${fmt(new Date(startsAt))}`,
     ].join("\n")
     sendTelegramMessage(tenantChatId, msg).catch(console.error)
   }
   ```

8. Return `NextResponse.json({ ok: true, booking: { startsAt, endsAt } })`.

Note: Access `telegramChatId` from the tenant include. The Tenant model has `telegramChatId` but it may not be in the TypeScript select type — cast as needed (same pattern used in `app/api/bookings/route.ts` line 168).

**Slot fetching effect**: When `rescheduleMode` is true and `selectedDate` changes, fetch slots:
```typescript
useEffect(() => {
  if (!rescheduleMode || !selectedDate) return
  setLoadingSlots(true)
  setSelectedSlot(null)
  fetch(`/api/bookings/slots?tenantSlug=${booking.tenantSlug}&resourceId=${booking.resourceId}&serviceId=${booking.serviceId}&date=${selectedDate}`)
    .then(res => res.json())
    .then(data => setSlots(data.slots ?? []))
    .catch(() => setSlots([]))
    .finally(() => setLoadingSlots(false))
```

```typescript
async function handleReschedule() {
  if (!selectedSlot) return
  setRescheduling(true)
  try {
    const res = await fetch(`/api/manage/${token}/reschedule`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ startsAt: selectedSlot.startsAt, endsAt: selectedSlot.endsAt }),
    })
    if (res.ok) {
      setRescheduled(true)
      setRescheduleMode(false)
    } else {
      const data = await res.json()
      setError(data.error || 'Ошибка при переносе')
    }
  } finally {
    setRescheduling(false)
  }
}
```
