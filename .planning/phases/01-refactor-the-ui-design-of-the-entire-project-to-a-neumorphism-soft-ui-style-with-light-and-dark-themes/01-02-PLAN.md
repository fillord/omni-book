- Keep the base class as-is (it already has `border border-transparent` which is correct for Neumorphism)
- Do NOT add `neu-btn` to the base class — it goes on specific variants only

**Variant changes:**

`default` variant — change from:
```
"bg-primary text-primary-foreground [a]:hover:bg-primary/80"
```
to:
```
"neu-btn text-foreground"
```
Rationale: `.neu-btn` provides the raised shadow, bg-color (var(--neu-bg)), hover accent color, and :active inset. `text-foreground` uses the Neumorphism text color. Remove `bg-primary` and `text-primary-foreground` — the button surface is now the same Neumorphism bg, not a solid primary color fill.

`outline` variant — change from:
```
"border-border bg-background hover:bg-muted hover:text-foreground aria-expanded:bg-muted aria-expanded:text-foreground dark:border-input dark:bg-input/30 dark:hover:bg-input/50"
```
to:
```
"neu-raised border-transparent text-foreground hover:text-neu-accent"
```
Rationale: Raised surface with transparent border. Remove all dark: overrides — Neumorphism tokens handle both themes.

`secondary` variant — change from:
```
"bg-secondary text-secondary-foreground hover:bg-secondary/80 aria-expanded:bg-secondary aria-expanded:text-secondary-foreground"
```
to:
```
"neu-raised bg-[var(--neu-bg)] text-secondary-foreground hover:text-neu-accent"
```

`ghost` variant — keep as-is BUT remove dark-specific override:
Change from:
```
"hover:bg-muted hover:text-foreground aria-expanded:bg-muted aria-expanded:text-foreground dark:hover:bg-muted/50"
```
to:
```
"hover:bg-muted hover:text-foreground aria-expanded:bg-muted aria-expanded:text-foreground"
```
(Remove `dark:hover:bg-muted/50` — token remapping handles dark mode.)

`destructive` variant — keep existing logic but simplify dark overrides:
Change from:
```
"bg-destructive/10 text-destructive hover:bg-destructive/20 focus-visible:border-destructive/40 focus-visible:ring-destructive/20 dark:bg-destructive/20 dark:hover:bg-destructive/30 dark:focus-visible:ring-destructive/40"
```
to:
```
"bg-destructive/10 text-destructive hover:bg-destructive/20 focus-visible:border-destructive/40 focus-visible:ring-destructive/20"
```
(Remove dark: overrides — destructive tokens handle both themes.)

`link` variant — keep unchanged: `"text-primary underline-offset-4 hover:underline"`
